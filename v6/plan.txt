specialized implementation: mission is exam score prediction using a nexus like architecture to see if it could be helpful

planned architecture:
 - preprocessing : processes dataset

 - judge : puts weights on different categories potentially based on category variables : distributes based on top weights
  -- note useful categories: age, gender, study hours, attendance, internet access, sleep hours, sleep quality, study method, facility rating, exam difficulty
  ---exam score is goal
  -- determines relevance of different features to find splitter priority (lowest 25% splitters are dropped for current input)
  -- will connect to splitter nodes to begin NN processing

 - splitter : takes data from judge and passes to closest 5% of processing nodes
  -- splitters are divided based on 2d quadrant or 3d octant
  -- currently no logic to how splitters get different values - judge manages that
  -- gets a feature and passes to processing nodes
  -- intent is generic routing as they create signals

 - processing : takes in some input and applies some kind of weight to it
  -- connections based on distance formula
  -- maybe bayes theorem or gauss
  -- the local modelto produce the prediciton and variance
  -- kills the signal if variance exceeds limits

 - reviewer : applies final weights based on possible nearest 5% of nodes return
  --located at furthest corners
  -- will attempt to reduce variance 
  -- akin to a critic which wont add weights but will stabilize final returns for each segment

 - handler : applies final weights from reviewers and returns answer while applying emphasis based on judge relevance scores
  -- final estimation node
  -- will utilize segment weights to find final values
  -- purely mathematical - no ML

 - signal : The representation of the data traveling between nodes 
    -- in a 2d system there should be 4 handlers and in total 4 * (.05 * total_processing_node_count) signals
    -- can be killed partway based on variance
    -- hop based weights to emphasize progression to end
    -- carries the position, segment weight, active prediction, accumulated variance, and signal life

Additional utilities:
 - dataset processing for training
 - training


final notes:
    It is definately a potential overkill to use a massive NN to do a prediction like this, but it will serve as the base test for predicitve capabilities of nexus architecture
    This is not intended to lead to generic models but a possible foundation for specialized models
    research purposes right now relying more on engineering rather than intense data usage
    hopefully more efficient than more advanced models

key notes to remember:
    - dont have nodes breach specializations
        -- handlers must not be trainable
        -- reviewers must not learn weights
        -- splitters must not be purely feature based
        -- positions are fundamental to nexus design
